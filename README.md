# 山勇ルーフ Webサイト納品ファイル

ロリポップ（共有サーバー）で動作するPHPサイト一式です。

## ディレクトリ構成

public_html/
├── assets/
│   ├── css/
│   │   └── style.css       # スタイルシート（デザイン設定）
│   ├── img/                # 画像フォルダ（ここに画像をアップロード）
│   └── js/                 # JavaScriptフォルダ
├── config/
│   └── config.php          # 設定ファイル（会社情報、メール設定）
├── partials/               # 共通パーツ
│   ├── head.php            # HTMLヘッダー（SEO, JSON-LD）
│   ├── header.php          # ページ上部ナビゲーション
│   └── footer.php          # ページ下部フッター
├── index.php               # トップページ
├── services.php            # 工事内容
├── works.php               # 施工事例
├── about.php               # 会社概要
├── contact.php             # お問い合わせフォーム
├── contact_confirm.php     # 確認画面
├── contact_send.php        # 送信処理
└── thanks.php              # 完了画面

## 導入手順

### 1. 設定の変更
`config/config.php` を開き、以下の部分を確認・変更してください。

```php
// 本番公開時はここを実際の受信メールアドレスに変更してください
define('MAIL_TO', 'info@example.com'); // ←ここを山本様のメールアドレスに

// 自動返信メールの送信元（レンタルサーバーのメールアドレス推奨）
// ロリポップで作成したメールアドレス（例: contact@sanyuroof.com）を設定すると迷惑メールに入りづらくなります
define('MAIL_FROM', 'noreply@' . $_SERVER['HTTP_HOST']); 
```

### 2. ファイルのアップロード
FTPソフト（FileZillaなど）を使って、`public_html` の中身を全てサーバーの公開ディレクトリ（通常は `public_html` やドメイン名のフォルダ）にアップロードしてください。

### 3. 画像の差し替え
`assets/img/` フォルダに以下の画像を配置すると、サイトの見栄えが良くなります。
- `hero-bg.jpg` : トップページのメイン画像
- `ogp.jpg` : SNSでシェアされた時の画像
- `logo.png` : 会社のロゴ

### 4. 動作確認
ブラウザでサイトにアクセスし、以下の点を確認してください。
- 表示崩れがないか
- お問い合わせフォームから送信して、指定したメールアドレスにメールが届くか
- スマホで見た時に「電話する」ボタンが下部に固定表示されるか

## ロリポップでメールが届かない場合

お問い合わせフォームから送信してもメールが届かない場合は、以下を確認してください。

1. **送信元アドレスの設定**
   `config.php` の `MAIL_FROM` に設定しているメールアドレスが、ロリポップで作成した独自ドメインのメールアドレスになっているか確認してください。Gmailなどを送信元にすると、なりすまし判定でブロックされることがあります。

2. **迷惑メールフォルダの確認**
   Gmailなどに転送している場合、迷惑メールフォルダに入っている可能性があります。

3. **PHPのバージョン**
   ロリポップの管理画面で、PHPのバージョンが `7.4` または `8.x` になっていることを確認してください（CGI版推奨）。

## 今後の運用について

- **施工事例を増やす**
  `works.php` のHTMLコードをコピーして、新しい事例を追加してください。写真はスマホで撮ったもので十分ですので、`assets/img/` にアップロードしてパスを指定してください。

- **お知らせなど**
  現在は静的な作りですが、将来的にWordpressなどを導入することも可能です。まずはこの構成で「問い合わせが来るか」をテストすることをお勧めします。
